## 5. Аутентификация и авторизация

### 5.1. Telegram Bot

**Механизм:** Токен бота через `TELEGRAM_BOT_TOKEN`

**Рекомендации:**
1. ✅ Использовать переменные окружения
2. ⚠️ Ограничить права бота в Telegram Bot Settings
3. ⚠️ Регулярно ротировать токен

### 5.2. ProxyAPI

**Механизм:** Bearer token через `PROXYAPI_API_KEY`

**Уязвимости:**
- ⚠️ Ключ виден только один раз при создании
- ⚠️ Нет механизма автоматической ротации

**Рекомендации:**
1. Хранить ключ в секрет-менеджере (Vault, AWS Secrets Manager)
2. Настроить мониторинг использования
3. Регулярно ротировать (каждые 90 дней)

### 5.3. GigaChat

**Механизм:** Authorization key (Basic auth)

**Рекомендации:**
1. Использовать OAuth flow вместо статического ключа
2. Кэшировать access token (уже реализовано)
3. Настроить автоматическое обновление
