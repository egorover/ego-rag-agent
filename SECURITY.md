# Политика безопасности

## Отчет об уязвимостях

Если вы обнаружили уязвимость, пожалуйста, сообщите нам через GitHub Issues.

## Безопасность данных

### API Ключи

- Все API ключи должны храниться в `.env` файле
- Файл `.env` добавлен в `.gitignore`
- Никогда не коммитьте реальные ключи в репозиторий
- Используйте `.env.example` как шаблон

### Переменные окружения

Обязательные переменные (зависят от `AI_PROVIDER`):
- `TELEGRAM_BOT_TOKEN` — токен Telegram бота (для запуска бота)
- `PROXYAPI_API_KEY` — при `AI_PROVIDER=proxyapi`
- `GIGACHAT_AUTHORIZATION_KEY` — при `AI_PROVIDER=gigachat`
- `OPENAI_API_KEY` — при `AI_PROVIDER=openai`

Эмбеддинги для RAG создаются **локально** (`sentence-transformers`). `OPENAI_API_KEY` не требуется при `gigachat` или `proxyapi`.
  - Получить на: https://console.proxyapi.ru/keys
  - ⚠️ Ключ можно увидеть полностью только один раз при создании!
  - Работает только с `https://api.proxyapi.ru`
- `GIGACHAT_AUTHORIZATION_KEY` — ключ GigaChat (при использовании)

### Рекомендации

1. Регулярно обновляйте API ключи
2. Используйте отдельные ключи для разработки и продакшена
3. Ограничьте права API ключей только необходимыми операциями
4. Мониторьте использование API ключей

## Безопасность сети

- Используйте HTTPS для всех API запросов
- При использовании прокси убедитесь в его безопасности
- Валидируйте SSL сертификаты в продакшене

## Ответственное раскрытие

Мы ценим вклад сообщества в безопасность проекта. При обнаружении уязвимости:

1. Создайте приватный issue на GitHub
2. Опишите уязвимость подробно
3. Мы ответим в течение 48 часов
4. После исправления будет опубликован CVE (если применимо)
